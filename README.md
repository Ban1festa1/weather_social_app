# Weather Social App

**Weather Social App** — итоговое Flutter-приложение с погодой, профилем пользователя, публикацией постов и Firebase Authentication.

Приложение позволяет пользователю зарегистрироваться или войти в аккаунт, посмотреть текущую погоду, создать пост с погодой и комментарием, просматривать список постов пользователей, открывать профили других пользователей, настраивать профиль и выходить из аккаунта.

## Реализованный функционал

* регистрация и вход пользователя через Firebase Authentication;
* выход из аккаунта на экране настроек;
* экран погоды с получением данных через HTTP-запрос;
* резервные погодные данные, если внешний API недоступен;
* создание поста с текущей погодой и комментарием;
* локальное сохранение созданных постов;
* отображение списка постов пользователей;
* открытие профиля другого пользователя;
* профиль текущего пользователя;
* выбор фотографии профиля через галерею;
* локальное сохранение аватара;
* модальные окна для выбора фото и подтверждения выхода;
* нижняя навигационная панель;
* собственная иконка приложения.

## Архитектура проекта

В проекте используется разделение на слои:

```text
lib/
 ├── data/
 │   ├── models/
 │   ├── repositories/
 │   └── services/
 │
 ├── presentation/
 │   ├── blocs/
 │   └── screens/
 │
 ├── firebase_options.dart
 └── main.dart
```

### Основные папки

`data/models` — модели данных приложения: пользователь, погода, пост.

`data/repositories` — репозитории для работы с пользователями, постами, профилем и погодой.

`data/services` — сервис для HTTP-запроса погоды.

`presentation/blocs` — BLoC для управления состоянием погоды.

`presentation/screens` — экраны приложения.

## Основные экраны

* экран авторизации;
* экран погоды;
* экран постов;
* экран профиля;
* экран настроек;
* экран создания поста;
* экран профиля другого пользователя.

## Запуск проекта

Перед запуском необходимо установить Flutter SDK и Android Studio.

Команды для запуска:

```bash
flutter pub get
flutter run
```

Для анализа проекта:

```bash
flutter analyze
```
